### Why

coderhub 目标是创建一个程序员分享生活动态的平台。当前项目是 Node.js + Koa + MySQL 后端，已有基础应用入口、用户路由、用户注册 controller/service，以及 `POST /users` 注册接口雏形。项目功能文档明确平台需要包含面向用户的业务接口，以及面向企业或内部的后台管理接口。

本规格定义 coderhub 第一版完整核心系统的功能行为，包括用户管理、内容管理、评论、标签、文件和后台管理接口。目标是让后续 design/coding 阶段能按明确范围拆分实现，不在编码阶段临时补需求。

| 文件管理 | 管理员可分页查看上传文件，并删除违规文件；被删除文件不能继续作为头像或内容图片展示 | P1 |
| 后台用户管理 | 管理员可分页查询用户、禁用用户、恢复用户；被禁用用户不能登录、发布内容、评论、上传文件 | P0 |
| 后台内容管理 | 管理员可分页查询全部状态内容，可审核、下架、删除内容；下架和删除内容不在公开列表展示 | P0 |
| 后台评论管理 | 管理员可分页查询评论，可删除违规评论；删除后的评论不在用户端评论列表展示 | P1 |

**不做的事：**

- 不做私信、关注、点赞、收藏、消息通知功能。
- 不做内容推荐算法、热榜排序、搜索引擎检索。
- 不做视频、音频、压缩包等多媒体文件上传。
- 不做第三方登录、短信登录、邮箱验证、找回密码。
- 不做多级后台组织、细粒度 RBAC 权限配置；本期只区分普通用户和管理员。
- 不做用户端自助创建标签；标签由后台统一维护。

### How

#### 1. 正常流程

1. 用户注册账号。
2. 用户登录成功后获得 token。
3. 用户携带 token 修改个人资料、上传头像或内容图片。
4. 管理员登录后台并维护可用标签。
5. 用户选择已启用标签，发布带文字和图片的生活动态。
6. 系统保存内容为待审核状态。
7. 管理员查看待审核内容并执行通过或驳回。
8. 审核通过后，内容进入公开列表，访客和用户可查看详情。
9. 已登录用户对公开内容发表评论或回复评论。
10. 作者、内容作者或管理员按权限删除评论。
11. 作者删除自己的内容，或管理员下架、删除违规内容。
12. 管理员分页管理用户、内容、评论、标签和文件。

#### 2. 关键场景

##### 场景：用户注册成功
- **Given** 用户名未被注册，且请求包含非空用户名和密码
- **When** 客户端提交注册请求
- **Then** 系统创建用户并返回 `code = 0` 的成功结果

##### 场景：重复用户名注册失败
- **Given** 用户名已存在
- **When** 客户端使用相同用户名提交注册请求
- **Then** 系统返回冲突错误，并且不创建新用户

##### 场景：用户登录成功
- **Given** 用户已注册、未被禁用，且密码正确
- **When** 客户端提交登录请求
- **Then** 系统返回 token，后续受保护接口可使用该 token 访问

##### 场景：无 token 访问受保护接口
- **Given** 客户端未携带 token
- **When** 客户端访问发布内容、评论、上传文件或后台管理接口
- **Then** 系统返回未认证错误，并且不执行写入操作

##### 场景：内容发布后等待审核
- **Given** 用户已登录，且选择的标签均为已启用标签
- **When** 用户发布生活动态
- **Then** 系统创建待审核内容，公开内容列表不展示该内容

##### 场景：内容审核通过后公开
- **Given** 管理员已登录，且存在待审核内容
- **When** 管理员审核通过该内容
- **Then** 公开内容列表和内容详情可查询到该内容

##### 场景：内容审核驳回
- **Given** 管理员已登录，且存在待审核内容
- **When** 管理员驳回该内容并填写驳回原因
- **Then** 内容状态变为已驳回，公开内容列表不展示该内容，作者可查看驳回结果

##### 场景：评论公开内容
- **Given** 用户已登录，且目标内容为已公开状态
- **When** 用户提交评论内容
- **Then** 系统创建评论，内容评论列表展示该评论

##### 场景：回复已删除评论失败
- **Given** 父评论不存在或已删除
- **When** 用户提交回复请求
- **Then** 系统返回目标不存在错误，并且不创建回复

##### 场景：禁用标签不可用于新内容
- **Given** 标签已被管理员禁用
- **When** 用户发布内容时选择该标签
- **Then** 系统返回参数错误，并且不创建内容

##### 场景：上传头像图片成功
- **Given** 用户已登录，且上传文件是符合限制的图片
- **When** 用户提交头像图片上传请求
- **Then** 系统返回文件访问信息，该文件可设置为用户头像

##### 场景：禁用用户访问失败
- **Given** 用户账号已被管理员禁用
- **When** 该用户登录或访问发布、评论、上传文件接口
- **Then** 系统返回账号禁用错误，并且不执行写入操作

#### 3. 失败处理

| 失败场景 | 处理方式 |
|---------|---------|
| 注册缺少用户名或密码 | 返回参数错误；不创建用户 |
| 注册用户名重复 | 返回冲突错误；不创建用户 |
| 登录用户名不存在 | 返回认证失败；不暴露用户是否存在的额外信息 |
| 登录密码错误 | 返回认证失败；不返回 token |
| 被禁用用户登录 | 返回账号禁用错误；不返回 token |
| 无 token 访问受保护接口 | 返回未认证错误；不执行目标操作 |
| token 无效或过期 | 返回未认证错误；客户端需要重新登录 |
| 普通用户访问后台接口 | 返回无权限错误；不返回后台数据 |
| 修改他人用户资料 | 返回无权限错误；不更新数据 |
| 发布内容缺少正文且无图片 | 返回参数错误；不创建内容 |
| 发布内容选择禁用或不存在标签 | 返回参数错误；不创建内容 |
| 查询不存在内容 | 返回目标不存在错误 |
| 公开访问待审核、已驳回、已下架、已删除内容 | 返回目标不存在错误或不在列表中出现 |
| 编辑他人内容 | 返回无权限错误；不更新内容 |
| 审核不存在或非待审核内容 | 返回状态错误；不改变内容状态 |
| 评论不存在内容 | 返回目标不存在错误；不创建评论 |
| 评论未公开内容 | 返回目标不存在错误或状态错误；不创建评论 |
| 回复不存在或已删除评论 | 返回目标不存在错误；不创建回复 |
| 删除无权限评论 | 返回无权限错误；不删除评论 |
| 创建重名标签 | 返回冲突错误；不创建标签 |
| 上传非图片文件 | 返回文件类型错误；不保存文件 |
| 上传超出大小限制图片 | 返回文件大小错误；不保存文件 |
| 使用已删除文件作为头像或内容图片 | 返回目标不存在错误；不更新引用 |
| 数据库写入失败 | 返回服务端错误；不返回半成功结果 |

#### 4. 边界与约束

| 边界 | 说明 |
|------|------|
| 用户角色 | 本期只区分普通用户和管理员两类角色 |
| token 适用范围 | 用户端写操作和全部后台管理接口必须校验 token |
| 内容状态 | 内容状态包含待审核、已公开、已驳回、已下架、已删除 |
| 内容可见性 | 只有已公开内容对访客和普通用户公开展示 |
| 作者可见性 | 作者可查看自己的待审核、已驳回和已公开内容 |
| 管理员可见性 | 管理员可查看全部状态内容 |
| 评论对象 | 评论和回复只能关联已公开且未删除内容 |
| 评论删除 | 删除评论后，该评论不在普通用户列表展示；其回复不再作为可回复目标 |
| 标签状态 | 标签包含启用和禁用；禁用标签不影响历史内容展示，但不能被新内容选择 |
| 文件类型 | 本期只支持头像图片和内容图片 |
| 文件引用 | 已删除文件不能继续作为头像或内容图片展示 |
| 分页 | 列表接口必须支持分页参数，并返回列表数据和总数信息 |
| 排序 | 公开内容、评论、后台列表默认按创建时间倒序返回 |
| 后台管理 | 后台管理接口只允许管理员访问 |

### Impact

- `src/main.js`：服务启动入口，后续功能仍通过统一 Koa app 启动。
- `src/app/index.js`：需要继续作为中间件和路由组装入口，承载用户、内容、评论、标签、文件、后台路由注册。
- `src/router/user_router.js`：已有用户路由，需要覆盖注册、登录、当前用户信息、资料维护相关接口。
- `src/controller/user_controller.js`：已有用户注册控制器，需要扩展用户相关请求处理行为。
- `src/service/user_service.js`：已有用户数据访问服务，需要覆盖注册、登录、查询、资料维护、禁用恢复等用户数据行为。
- `src/app/database.js`：现有 MySQL 连接池由各业务服务复用。
- 新增内容、评论、标签、文件、后台管理相关模块：用于承载本 spec 中除用户外的核心系统行为。
